# CLI リファレンス

Rust バイナリ `waragent` は 3 つのサブコマンド `run` / `sweep` / `reproduce` を持つ．

## `waragent run`

LLM 駆動の国エージェント外交 ABM を単一設定で実行する．

| オプション | 既定 | 意味 |
|---|---|---|
| `--scenario <S>` | `wwi` | `wwi` (8 カ国) または `wwi-small` (4 カ国・軽量) |
| `--trigger <T>` | `archduke-assassination` | `null` / `naval-incident` / `dardanelles` / `archduke-assassination` (round 0 イベント) |
| `--stance <S>` | (シナリオ既定) | 全国のスタンスを上書き: `conservative` / `neutral` / `aggressive` |
| `--secretary-passes <N>` | `1` | LLM 秘書検証パス数; 1 ラウンドの LLM 呼び出し = `国数 × (1 + N)` |
| `--rounds <N>` | `6` | 最大ラウンド数 (論文評価は round 6) |
| `--war-threshold <N>` | `3` | 世界大戦勃発とみなす宣戦 (W) 対の数 (run を停止) |
| `--runs <N>` | `1` | 独立試行数 (各試行は独立コア seed を派生) |
| `--seed <N>` | (ランダム) | 基点 RNG seed (socsim コアのみ支配) |
| `--llm-temperature <F>` | `0.0` | 生成温度 (再現性のため 0) |
| `--llm-seed <N>` | `0` | バックエンドへ渡す生成 seed |
| `--cache-path <P>` | `.llm_cache/cache.json` | プロンプト→応答キャッシュファイル |
| `--output-dir <D>` | `results` | 出力ベースディレクトリ |

例 (小さな実 LLM スモーク):

```bash
cargo run --release -- run --scenario wwi-small --rounds 2 --runs 1 --secretary-passes 1 --seed 42
```

## `waragent sweep`

トリガー × スタンス を走査し，開戦率と同盟 MI を集計する．

| オプション | 既定 | 意味 |
|---|---|---|
| `--scenario <S>` | `wwi` | シナリオ |
| `--trigger-values <CSV>` | `null,naval-incident,dardanelles` | カンマ区切りトリガー候補 |
| `--stance-values <CSV>` | `conservative,aggressive` | カンマ区切りスタンス候補 |
| `--secretary-passes <N>` | `1` | 上記と同じ |
| `--rounds <N>` | `6` | 最大ラウンド数 |
| `--war-threshold <N>` | `3` | 上記と同じ |
| `--runs <N>` | `3` | (trigger, stance) セルあたりの試行数 |
| `--seed <N>` | `42` | 基点 seed (セル × 試行ごとに派生) |
| `--cache-path <P>` | `.llm_cache/cache.json` | 共有キャッシュ (スイープ全体でヒット率を高める) |
| `--output-dir <D>` | `results` | 出力ベースディレクトリ |

例:

```bash
cargo run --release -- sweep \
    --scenario wwi \
    --trigger-values null,naval-incident,dardanelles \
    --stance-values conservative,aggressive \
    --rounds 6 --runs 7 --seed 42
```

## `waragent reproduce`

論文の Table 2–5 ヘッドライン story — トリガー強度に応じた開戦頻度・同盟エスカレーション・同盟分極化 — を 3 つのトリガー条件 (`null` / `dardanelles` / `archduke-assassination`) で軽量シナリオ上に走らせ，観測値 vs 論文値のアンカー (PASS/off) と figure 入力を `reproduce_summary.json` に集約する．

| オプション | 既定 | 意味 |
|---|---|---|
| `--scenario <S>` | `wwi-small` | シナリオ (軽量シナリオでバンドルをオフライン検証可能に保つ) |
| `--secretary-passes <N>` | `1` | LLM 秘書検証パス数 |
| `--rounds <N>` | `6` | 各トリガー条件の最大ラウンド数 (`--quick` で 2 に縮約) |
| `--war-threshold <N>` | `2` | 開戦とみなす宣戦布告 (W) 対数 |
| `--seed <N>` | `42` | シード基点 (トリガー条件ごとに派生) |
| `--llm-temperature <F>` | `0.0` | 生成温度 |
| `--llm-seed <N>` | `0` | 生成シード |
| `--cache-path <P>` | `.llm_cache/cache.json` | プロンプト→応答キャッシュ (ライブ経路のみ; mock は in-memory) |
| `--output-dir <D>` | `results` | 出力ベースディレクトリ |
| `--mock` | off | ライブ LLM の代わりにトリガー感応な scripted mock を使う (オフライン / CI) |
| `--quick` | off | 短縮再現 (`rounds=2`) |

例 (オフライン・ライブ LLM 不要):

```bash
cargo run --release -- reproduce --mock --quick
```

`results/{timestamp}_reproduce/` に `reproduce_summary.json` と，トリガー条件ごとのサブディレクトリ (`metrics.csv` / `events.csv` / `run_metadata.json` / `config.json`) を書き出す．図は `uv run waragent-tools reproduce` で描画する (`--run --mock --quick` で先にバイナリを実行可能)．

## オフライン mock スモーク

```bash
cargo run --release --example mock_smoke -- results
```
