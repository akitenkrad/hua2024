# CLI reference

The Rust binary `waragent` has three subcommands: `run`, `sweep`, and `reproduce`.

## `waragent run`

Runs a single configuration of the LLM-driven country-agent diplomacy ABM.

| Option | Default | Meaning |
|---|---|---|
| `--scenario <S>` | `wwi` | `wwi` (8 countries) or `wwi-small` (4 countries, light) |
| `--trigger <T>` | `archduke-assassination` | `null` / `naval-incident` / `dardanelles` / `archduke-assassination` (round-0 event) |
| `--stance <S>` | (scenario default) | override every country's stance: `conservative` / `neutral` / `aggressive` |
| `--secretary-passes <N>` | `1` | LLM secretary verification passes; LLM calls per round = `n_countries × (1 + N)` |
| `--rounds <N>` | `6` | maximum rounds (paper evaluates at round 6) |
| `--war-threshold <N>` | `3` | number of war (W) pairs that counts as world-war outbreak (stops the run) |
| `--runs <N>` | `1` | independent trials (each derives an independent core seed) |
| `--seed <N>` | (random) | base RNG seed (governs the socsim core only) |
| `--llm-temperature <F>` | `0.0` | generation temperature (kept 0 for reproducibility) |
| `--llm-seed <N>` | `0` | generation seed passed to the backend |
| `--cache-path <P>` | `.llm_cache/cache.json` | prompt→response cache file |
| `--output-dir <D>` | `results` | output base directory |

Example (small live smoke):

```bash
cargo run --release -- run --scenario wwi-small --rounds 2 --runs 1 --secretary-passes 1 --seed 42
```

## `waragent sweep`

Sweeps trigger × stance and aggregates war-outbreak rate and alliance MI.

| Option | Default | Meaning |
|---|---|---|
| `--scenario <S>` | `wwi` | scenario |
| `--trigger-values <CSV>` | `null,naval-incident,dardanelles` | comma-separated trigger candidates |
| `--stance-values <CSV>` | `conservative,aggressive` | comma-separated stance candidates |
| `--secretary-passes <N>` | `1` | as above |
| `--rounds <N>` | `6` | max rounds |
| `--war-threshold <N>` | `3` | as above |
| `--runs <N>` | `3` | trials per (trigger, stance) cell |
| `--seed <N>` | `42` | base seed (derived per cell × trial) |
| `--cache-path <P>` | `.llm_cache/cache.json` | shared cache (maximises hit rate across the sweep) |
| `--output-dir <D>` | `results` | output base directory |

Example:

```bash
cargo run --release -- sweep \
    --scenario wwi \
    --trigger-values null,naval-incident,dardanelles \
    --stance-values conservative,aggressive \
    --rounds 6 --runs 7 --seed 42
```

## `waragent reproduce`

Reproduces the paper's Table 2–5 headline story — trigger-intensity-dependent war-outbreak frequency, alliance escalation, and alliance polarization — by running three trigger conditions (`null` / `dardanelles` / `archduke-assassination`) on a light scenario and aggregating observed-vs-paper anchors (PASS/off) plus figure inputs into `reproduce_summary.json`.

| Option | Default | Meaning |
|---|---|---|
| `--scenario <S>` | `wwi-small` | scenario (light scenario keeps the bundle offline-verifiable) |
| `--secretary-passes <N>` | `1` | LLM secretary verification passes |
| `--rounds <N>` | `6` | max rounds per trigger condition (`--quick` shrinks to 2) |
| `--war-threshold <N>` | `2` | war (W) pairs that count as outbreak |
| `--seed <N>` | `42` | base seed (derived per trigger condition) |
| `--llm-temperature <F>` | `0.0` | generation temperature |
| `--llm-seed <N>` | `0` | generation seed |
| `--cache-path <P>` | `.llm_cache/cache.json` | prompt→response cache (live path only; mock uses in-memory) |
| `--output-dir <D>` | `results` | output base directory |
| `--mock` | off | use a scripted, trigger-sensitive mock instead of a live LLM (offline / CI) |
| `--quick` | off | short reproduction (`rounds=2`) |

Example (offline, no live LLM):

```bash
cargo run --release -- reproduce --mock --quick
```

Writes `results/{timestamp}_reproduce/` with `reproduce_summary.json` and one subdirectory per trigger condition (`metrics.csv` / `events.csv` / `run_metadata.json` / `config.json`). Render the figures with `uv run waragent-tools reproduce` (add `--run --mock --quick` to run the binary first).

## Offline mock smoke

```bash
cargo run --release --example mock_smoke -- results
```

Drives a scripted mock client (no live LLM) over a 4-country / 2-round scenario and writes the same outputs as `run`.
