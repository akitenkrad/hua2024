## 二層決定論

socsim コアは決定論的かつ LLM 非依存だが，LLM は本質的に非決定的である．本設計は LLM を 1 メカニズムに閉じ込め擬似決定論化する:

| レイヤ | 構成要素 | 再現性 |
|---|---|---|
| 決定論的 socsim コア | シナリオ/Board 初期化・活性化順・publicity 伝播・同盟/宣戦/条約の解決・エスカレーション・Board と総動員の更新・全指標 | seed から bit 単位で再現 (ChaCha20 `SimRng`) |
| 非決定的 LLM レイヤ | `CountryDecisionMechanism` のみ (4 ステップ推論 + 秘書検証) | `socsim-llm` キャッシュ + `temperature=0` + 固定 seed で擬似決定論 |

RNG ストリーム: 世界/シナリオ初期化に `derive_seed(root, &[0])`，エンジン (活性化順) に `derive_seed(root, &[1])`．`&[2..]` は予約．

## メカニズム (6 フェーズ上の 5 つ)

| メカニズム | フェーズ | 役割 |
|---|---|---|
| `SituationMechanism` | Environment | 各国の Board/Stick を文脈パラグラフへ Translate; round 0 にトリガー注入 |
| `CountryDecisionMechanism` | **Decision** | 唯一の LLM 所在: 各国が 4 ステップ誘導推論 + `secretary_passes` 回の検証を行い，1 行動を `pending_actions` へ蓄積 |
| `DiplomacyMechanism` | Interaction | publicity 伝播 (公開=ブロードキャスト / 秘密=対象のみ); 宣戦・相互同盟/不干渉・和平を解決し Board を更新 |
| `EscalationMechanism` | Interaction | エスカレーション動学 — 宣戦された側の同盟国が攻撃国に自動参戦 (1 ホップ) |
| `BoardUpdateMechanism` | PostStep | 総動員 (Stick MO) 確定・イベントログ追記・ラウンド指標計算・開戦 (W 対 ≥ `war_threshold`) または最終ラウンドで停止 |

更新セマンティクスは **同期的**: 全国が `Decision` で決定し (Board 不変)，蓄積した `pending_actions` を `Interaction` でまとめて解決する．1 tick = 1 ラウンド．スケジューラは `RandomActivationScheduler`．

## 関係構造

各国固有の `Board` (`(owner, other) → W/M/T/P`; 明示的 `BTreeMap`) を source of truth とする (部分情報の原則)．`metrics::alliance_partition` は同盟 (M) エッジ上の union-find で同盟クラスタを導出し，`metrics::alliance_network` は同等のグローバル無向 `socsim-net::SocialNetwork` を構築する．後者の連結成分数は分割と一致する (相互検算)．空間格子は使わない (`socsim-grid` 非依存)．

## 指標

- `alliance_mi` — シミュレート同盟分割と史実 WWI 分割 (`{A,B,G}`, `{C,D,E,H}`, `{F}`) の正規化相互情報量 (`[0,1]`)．
- `declaration_jaccard` — シミュレート宣戦 (W) 対集合と史実集合の Jaccard．
- `mobilization_jaccard` — シミュレート総動員国集合と史実集合の Jaccard．
- マクロフラグ: `war_outbreak`, `escalation_round`, `n_conflicts`, `cold_war_flag`．

史実参照集合は `metrics.rs` の匿名 id 定数．MI/Jaccard は Rust 側で計算する (scipy 非依存)．
